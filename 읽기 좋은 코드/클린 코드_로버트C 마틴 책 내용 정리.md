# 2장 의미 있는 이름

**의도를 분명히 밝혀라**
- 변수나 함수 그리고 클래스 이름은 존재 이유, 수행 기능, 사용 방법에 대한 질문에 모두 답을 할 수 있어야 한다.
- 주석이 필요하다면 의도를 분명히 드러내지 못했다는 말이다.

**발음하기 쉬운 이름을 사용하라**
- genymdhms(generate date, year, month, day, hour, minute, second)라는 변수를 사용하면, 발음이 되지 않는 변수명 때문에 코드를 작성할 때도 난처해지고, 동료들끼리 의사소통 하기에도 어렵다.
- genymdhms -> generationTimestamp 라고 지으면 변수명이 명확해진다.

**한 개념에 한 단어를 사용하라**
- 추상적인 개념 하나에 단어 하나를 선택해 이를 고수한다. ex) 조회 메서드의 접두사로 find는 nullable한 값이 반환될 수 있음을 의미하고, get은 반환 값이 NotNull임을 의미
- 동일 코드 기반에 Controller, Manager, Driver를 섞어 쓰면 혼란스럽다. 일관성있게 지어야 한다.
